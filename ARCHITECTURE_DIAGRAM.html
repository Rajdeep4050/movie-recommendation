🏗️ System Architecture Layers & Components
👤 User Interaction Layer

Django frontend (index.html) - Users search for movies via web browser. Handles GET/POST requests, form submission, displays recommendations in result.html.

🔗 Web Application Layer

Django 6.0 backend processes requests, Advanced Fuzzy Matching (Levenshtein distance) finds user's movie even with typos, Health Check validates model status. Supports separate movie recommendation and genre discovery modes. Runs on 0.0.0.0:8000.

🤖 ML Inference Engine

TF-IDF Vectorizer converts text features to sparse vectors (20K dimensions). SVD Reducer reduces to 300 components (85% variance). Cosine Similarity performs O(1) lookups on precomputed scores.

💾 Model Storage (145 MB)

similarity_matrix.npz (145MB) - Sparse 6248×6248 matrix. movie_metadata (1.1MB) - 6248 movies in Parquet. title_to_idx.json - Title→index mapping. Vectorizers (410KB + 24MB) - Trained TF-IDF & SVD models.

📅 Airflow MLOps Pipeline

Apache Airflow 2.10.4 orchestrates 10 tasks weekly: Check Credentials → Download TMDB (1.4M movies) → Validate Data → Train Model (15-20min) → Evaluate → Deploy → Backup → Notify → Restart Django.

📊 Data Source

Kaggle TMDB API provides 1.4M movies with 30+ features (genres, keywords, ratings, cast, production companies). 30-day cache prevents redundant downloads.

⚡ Performance Characteristics

Request latency: <200ms (Levenshtein fuzzy match + similarity lookup). Model load: ~2 seconds (background thread). Weekly retrain: 15-20 minutes. Model accuracy: Evaluated with Recall@15 and NDCG metrics. Fuzzy matching: Handles typos and partial matches with intelligent scoring.

🔄 Request Flow

User selects search mode (movie recommendations or genre discovery) → Django processes request → For movie search: fuzzy-matches title with Levenshtein distance → Gets movie index → Loads precomputed similarities → Returns top 15 + metadata. For genre search: filters by genre and sorts by rating. All data cached in memory for instant response.
